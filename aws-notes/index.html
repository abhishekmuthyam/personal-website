Serverless — what this site runs on

Lambda ● live on this site

Run code without servers: upload a function, AWS runs it per-request and bills per millisecond. Always-free tier: 1M requests + 400,000 GB-seconds per month, forever. This site's game leaderboard and YouTube search proxy are both Lambdas (Node.js 22, 128 MB, ap-south-1).

  • Cold start: the first request after idle spins a new sandbox (~100–400ms for small Node functions) — fine for hobby APIs; tune with provisioned concurrency if it ever matters.
  • Keep functions single-purpose; environment variables hold secrets (our YouTube API key lives there, never in frontend code).

Lambda Function URLs vs API Gateway

A Function URL is a free, built-in HTTPS endpoint directly on a Lambda — no extra service. API Gateway adds throttling, API keys, custom domains, and request validation, but costs ~$1/million calls after the free year. For a personal site, Function URLs + CORS config cover everything at $0.

Hard-won gotcha: creating a public (auth NONE) Function URL via the CLI needs two resource-policy permissions — lambda:InvokeFunctionUrl and lambda:InvokeFunction. The console adds both silently; the CLI does not, and you get a mysterious 403 Forbidden until the second one exists.

aws lambda add-permission --function-name fn \
  --action lambda:InvokeFunctionUrl --principal "*" \
  --function-url-auth-type NONE --statement-id url
aws lambda add-permission --function-name fn \
  --action lambda:InvokeFunction --principal "*" \
  --statement-id invoke

DynamoDB ● live on this site

Serverless NoSQL key-value store. You model around access patterns, not normalized tables: choose a partition key (and optional sort key) so every query is a direct lookup. The leaderboard table uses game as partition key and a zero-padded scoreKey sort key, so "top 10 scores" is a single Query with ScanIndexForward=false, Limit=10 — no sorting in code.

  • Always-free: 25 GB storage + 25 provisioned RCU/WCU. Use provisioned mode to stay free — on-demand mode bills per request.
  • TTL: set an epoch-seconds attribute and DynamoDB deletes expired items for free — our YouTube search cache expires entries after 24h this way.
  • Query reads one partition efficiently; Scan reads the whole table — avoid it in request paths.

Cognito ● live on this site

Managed user directory: signup, email verification, login, token refresh — you never store passwords. This site's login is a Cognito User Pool called directly from browser JavaScript via its REST API (SignUp, ConfirmSignUp, InitiateAuth) — no SDK needed.

  • Successful login returns JWTs: an ID token (who the user is — name, email in claims) and a refresh token (silently get new ID tokens; this is how "stay logged in" works).
  • Lite tier is free to 50k monthly active users.
  • Built-in email sender caps at ~50 mails/day — wire SES when a real user base shows up.

Core services worth knowing cold

IAM — identity and access management

Everything in AWS is denied unless a policy allows it. Users/roles get policies; services assume roles. Rules that keep you safe:

  • Never use the root account day-to-day — create an IAM user/role even for yourself.
  • Least privilege: our leaderboard Lambda's role can only Query/PutItem on one specific table ARN — nothing else.
  • Roles over access keys wherever possible; for Lambdas the execution role is automatic and keyless.

S3 — object storage

Buckets hold objects (files) addressed by key; famously durable. Use cases: static website hosting, backups, data lakes, user uploads. Know the storage classes (Standard → Infrequent Access → Glacier) and that lifecycle rules move objects between them automatically. Public access is blocked by default now — pre-signed URLs are the right way to grant temporary access to private objects.

EC2 — virtual machines

Full servers for long-running processes, custom runtimes, or stateful software. Interview basics: instance types (t-family burstable for dev), security groups are stateful firewalls, EBS volumes persist independently of the instance, and an Auto Scaling Group behind a load balancer is the classic high-availability pattern. For this site's workloads Lambda beats EC2: nothing to patch and no idle cost.

CloudWatch & Budgets

CloudWatch collects logs and metrics — every Lambda's console.log lands in a log group automatically (that's how you debug them). Alarms fire on metric thresholds. Separately, AWS Budgets emails you when actual or forecast spend crosses a limit — the first thing to set up on any new account. This account runs a $1/month budget with alerts at 50% and 100%; the always-free design means it should never fire.

Free tier, straight talk (2026)

What's actually free, forever?

  • New accounts (after July 2025): a Free plan — up to $200 credits for 6 months, cannot be charged, auto-closes unless upgraded. Upgrading to the paid plan has no monthly fee; it's pay-as-you-go.
  • Always-free regardless of plan: Lambda (1M requests/month), DynamoDB (25 GB + 25 provisioned RCU/WCU), Cognito Lite (50k MAU), CloudWatch basics, small SNS/SQS allowances.
  • Not always-free (watch these): API Gateway, NAT Gateways, EC2, RDS, on-demand DynamoDB, and data transfer beyond the monthly allowance.

Designed carefully — Function URLs instead of API Gateway, provisioned instead of on-demand — a personal site's entire backend runs at $0.00/month indefinitely. This one does.
